📈 情境 4：數據分析與建議

你說：

我的網站有 5000 訪客、300 註冊、50 付費，每人消費 800 元，幫我分析

龍蝦會回覆：

📈 網站數據分析報告

【關鍵指標】
- 總營收：50 × 800 = 40,000 元
- 訪客→註冊：300÷5000 = 6% ✅ 良好（業界 2-5%）
- 註冊→付費：50÷300 = 16.7% ✅ 優秀（業界 5-10%）
- 整體轉換：50÷5000 = 1%

【發現】
✅ 產品有吸引力（註冊到付費轉換高）
⚠️ 訪客到註冊可再提升

【建議】
1. 優化註冊頁面，減少填寫欄位
2. 加入社群帳號一鍵註冊
3. 提供首次註冊優惠

🔍 情境 10：程式碼審查

你說：

幫我看這段程式碼有沒有問題：
query = "SELECT * FROM users WHERE username='" + username + "'"

龍蝦會回覆：

🚨 嚴重安全漏洞！SQL Injection

【問題】
這段程式碼容易被 SQL 注入攻擊！
駭客輸入 ' OR '1'='1 就能繞過驗證

【正確寫法】
# 使用參數化查詢
query = "SELECT * FROM users WHERE username = %s"
cursor.execute(query, (username,))

【額外建議】
1. 密碼不要存明碼，用 bcrypt 加密
2. 加入登入失敗次數限制
3. 使用 HTTPS 保護傳輸

⚠️ 這種漏洞可能讓整個資料庫被偷走！

🤝 情境 15：商業談判策略

你說：

供應商要漲價 20%，幫我想談判策略

龍蝦會回覆：

🤝 供應商漲價談判策略

【準備工作】
1. 調查市場行情，了解其他供應商報價
2. 計算漲價對你的成本影響
3. 準備 B 方案（備選供應商）

【談判策略】
1. 先傾聽對方漲價原因
2. 表達長期合作誠意
3. 提出替代方案：
   - 接受 10% 漲幅（折衷）
   - 增加訂購量換取原價
   - 延長付款期限換取較低漲幅

【談判話術】
「我們合作這麼久，希望能找到雙贏的方案。
如果維持原價，我們可以承諾下半年增加 30% 訂單。」

💡 永遠要有備選方案，才有談判籌碼
