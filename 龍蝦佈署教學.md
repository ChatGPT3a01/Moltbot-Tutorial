## 1. 什麼是 Clawdbot/Moltbot？

Clawdbot/Moltbot（也叫 Moltbot）是一隻 **AI 聊天機器人**。

它可以：
- 回答你的問題
- 幫你查資料
- 透過 LINE 跟你聊天
- 幫你做一些事情

它就像一個住在電腦裡的小助手！

---

## 2. 安裝前要準備的東西

開始之前，請先確認你有：

| 項目 | 說明 |
|------|------|
| Windows 11 電腦 | 你的電腦 |
| 網路連線 | 要能上網 |
| LINE 帳號 | 用來跟機器人聊天 |
| ChatGPT 帳號 | 機器人的大腦（有 Plus 訂閱更好） |

## 3. 第一步：安裝 nvm

nvm 是一個工具，可以幫你管理 Node.js 的版本。

### 3.1 下載 nvm

1. 打開瀏覽器
2. 進入這個網址：
   ```
   https://github.com/coreybutler/nvm-windows/releases
   ```
3. 往下滑，找到 **Assets**
4. 點 **nvm-setup.exe** 下載

### 3.2 安裝 nvm

1. 打開剛剛下載的 `nvm-setup.exe`
2. 一直按 **Next**（下一步）
3. 按 **Install**（安裝）
4. 等它跑完，按 **Finish**（完成）

### 3.3 確認安裝成功

1. 按下鍵盤的 `Windows 鍵 + R`
2. 輸入 `powershell`，按 Enter
3. 在 PowerShell 視窗輸入：
   ```powershell
   nvm version
   ```
4. 如果看到版本號（例如 `1.2.2`），就代表成功了！

---

## 4. 第二步：安裝 Node.js

Node.js 是讓 Clawdbot/Moltbot 能夠運作的程式。

### 4.1 打開 PowerShell（用系統管理員）

1. 點 **開始**（左下角的 Windows 圖示）
2. 搜尋 `PowerShell`
3. 在 **Windows PowerShell** 上面 **按右鍵**
4. 選 **以系統管理員身分執行**
5. 如果跳出詢問視窗，按 **是**

### 4.2 安裝 Node.js 24 版

在 PowerShell 視窗輸入這行，然後按 Enter：

```powershell
nvm install 24
```

等它下載完成，會顯示 `Installation complete`。

### 4.3 切換到 Node.js 24 版

輸入這行，按 Enter：

```powershell
nvm use 24
```

會顯示 `Now using node v24.x.x`。

### 4.4 確認安裝成功

輸入這兩行（一行一行輸入，每輸入一行按一次 Enter）：

```powershell
node -v
npm -v
```

你應該看到類似這樣：
```
v24.13.0
11.6.4
```

看到版本號就代表成功了！

## 5. 第三步：安裝 Clawdbot/Moltbot

### 5.1 建立工作資料夾

在 PowerShell 輸入：

```powershell
mkdir D:\Moltbot
cd D:\Moltbot
```

### 5.2 執行安裝指令

輸入這行：

```powershell
iwr -useb https://molt.bot/install.ps1 | iex
```

等它跑，會出現一個有龍蝦圖案的畫面！

### 5.3 回答安裝問題

安裝過程會問你很多問題，照著下面選：

---

用方向鍵 ↑↓ 選 **Yes**，按 Enter

---

**問題 2：Onboarding mode**

選 **QuickStart**，按 Enter

---

**問題 3：Model/auth provider**

選 **OpenAI (Codex OAuth + API key)**，按 Enter

---

**問題 4：OpenAI auth method**

選 **OpenAI Codex OAuth (ChatGPT OAuth)**，按 Enter

這時候瀏覽器會自動打開，要你登入 ChatGPT。

1. 登入你的 ChatGPT 帳號
2. 登入完成後，網頁會跳轉
3. 回到 PowerShell，它會自動繼續

---

**問題 5：Default model**

選 **Keep current (openai-codex/gpt-5.2)**，按 Enter

---

**問題 6：Select channel (QuickStart)**

用方向鍵找到 **LINE (Messaging API)**，按 Enter

---

**問題 7：Install LINE plugin?**

選 **Skip for now**，按 Enter

（之後會手動設定）

---

**問題 8：Configure skills now?**

選 **No**，按 Enter

---

**問題 9：Enable hooks?**

按 **空白鍵** 選中 **Skip for now**，然後按 Enter

看到這些訊息就代表成功：

```

```

## 6. 第四步：建立 LINE 機器人帳號

### 6.1 進入 LINE Developers

1. 打開瀏覽器
2. 進入：`https://developers.line.biz/`
3. 點右上角 **Log in**
4. 用你的 LINE 帳號登入

### 6.2 建立 Provider

1. 登入後，點 **Create**（或 **建立**）
2. 輸入 Provider 名稱（例如：`我的機器人`）
3. 點 **Create**

### 6.3 建立 Messaging API Channel

1. 點 **Create a Messaging API channel**
2. 填寫資料：
   - **Channel name**：機器人的名字（隨便取）
   - **Channel description**：機器人的說明（隨便寫）
   - **Category**：選一個類別
   - **Subcategory**：選一個子類別
3. 勾選最下面的同意條款 ☑
4. 點 **Create**

### 6.4 取得 Channel Secret

1. 進入你剛建立的 Channel
2. 點上面的 **Basic settings**
3. 找到 **Channel secret**
4. 點旁邊的 **複製** 按鈕
5. 打開記事本，貼上，寫上「Channel Secret:」

### 6.5 取得 Channel Access Token

1. 點上面的 **Messaging API**
2. 往下滑，找到 **Channel access token (long-lived)**
3. 點 **Issue**（產生）
4. 會出現一長串文字
5. 點 **複製** 按鈕
6. 貼到記事本，寫上「Channel Access Token:」

### 6.6 取得 Channel ID

1. 回到 **Basic settings**
2. 找到 **Channel ID**
3. 複製下來，貼到記事本，寫上「Channel ID:」

現在你的記事本應該有三行：
```
Channel ID: 1234567890
Channel Secret: abcdef123456
Channel Access Token: 很長一串英文數字...
```

**這三個值很重要，等等要用！**

## 7. 第五步：安裝 ngrok

ngrok 可以讓 LINE 的伺服器連到你的電腦。

### 7.1 安裝 ngrok

在 PowerShell 輸入：

```powershell
winget install ngrok.ngrok
```

等它跑完，會顯示「已成功安裝」。

### 7.2 註冊 ngrok 帳號

1. 打開瀏覽器
2. 進入：`https://dashboard.ngrok.com/signup`
3. 可以用 Google 帳號註冊（比較快）
4. 登入後，點左邊的 **Your Authtoken**
5. 看到一串很長的文字，點旁邊的 **複製** 按鈕

### 7.3 設定 ngrok

**重要：關掉 PowerShell，重新開一個新的！**

1. 點 **開始**
2. 搜尋 `PowerShell`
3. 開啟 **Windows PowerShell**

在新的 PowerShell 輸入（把 `你的token` 換成剛剛複製的那串）：

```powershell
ngrok config add-authtoken 你的token
```

會顯示 `Authtoken saved to configuration file`。

### 7.4 如果 ngrok 設定檔出錯

如果看到 `unknown version '3'` 的錯誤：

1. 按 `Windows 鍵 + R`
2. 輸入 `notepad`，按 Enter，打開記事本
3. 在記事本點 **檔案** → **開啟**
4. 在檔案名稱輸入：
   ```
   C:\Users\你的使用者名稱\AppData\Local\ngrok\ngrok.yml
   ```
   （把「你的使用者名稱」換成你電腦的使用者名稱）
5. 把內容改成：
   ```yaml
   version: "2"
   authtoken: 你的ngrok_token
   ```
6. 存檔

### 7.5 更新 ngrok（如果需要）

如果看到版本太舊的錯誤，輸入：

```powershell
ngrok update
```

## 8. 第六步：設定 LINE Webhook

### 8.1 修改 Clawdbot/Moltbot 設定檔

1. 按 `Windows 鍵 + R`
2. 輸入 `notepad`，按 Enter
3. 在記事本點 **檔案** → **開啟**
4. 在檔案名稱輸入：
   ```
   C:\Users\你的使用者名稱\.clawdbot\clawdbot.json
   ```
5. 找到這一段（可能在最後面）：
   ```json
   "channels": {
   ```
6. 如果沒有 `channels`，找一個 `}` 的前面，加入：
   ```json
   "channels": {
     "line": {
       "enabled": true,
       "channelId": "你的Channel ID",
       "channelSecret": "你的Channel Secret",
       "channelAccessToken": "你的Channel Access Token"
     }
   },
   ```
7. 把剛剛記事本記下來的三個值填進去
8. 存檔

### 8.2 重啟 Clawdbot/Moltbot

在 PowerShell 輸入：

```powershell
clawdbot gateway restart
```

### 8.3 啟動 ngrok

**開一個新的 PowerShell 視窗**（不要關掉舊的），輸入：

```powershell
ngrok http 18789
```

你會看到一個表格，找到 **Forwarding** 那一行，例如：

```
Forwarding    https://abcd1234.ngrok-free.app -> http://localhost:18789
```

把 `https://abcd1234.ngrok-free.app` 這個網址複製下來！

**注意：不要關掉這個視窗，ngrok 要一直開著！**

### 8.4 設定 LINE Webhook URL

1. 回到 LINE Developers 網站
2. 進入你的 Channel
3. 點 **Messaging API**
4. 找到 **Webhook URL**
5. 點 **Edit**
6. 在輸入框貼上 ngrok 的網址，後面加上 `/line/webhook`，例如：
   ```
   https://abcd1234.ngrok-free.app/line/webhook
   ```
7. 點 **Update**
8. 把 **Use webhook** 的開關打開（變成綠色的 ON）

---

## 9. 第七步：配對你的 LINE 帳號

### 9.1 加機器人好友

1. 在 LINE Developers 的 **Messaging API** 頁面
2. 找到 **QR code**
3. 用手機打開 LINE
4. 點右上角的 **加好友**
5. 點 **行動條碼**
6. 掃描電腦上的 QR code
7. 加機器人為好友

### 9.2 傳訊息給機器人

用手機 LINE 對機器人說：

```
你好
```

### 9.3 機器人會回覆配對碼

機器人會回覆類似這樣的訊息：

```
Clawdbot/Moltbot: access not configured.
Your lineUserId: Uxxxxxxxxxxxxxxxx
Pairing code: ABC123
Ask the bot owner to approve with:
clawdbot pairing approve line <code>
```

記下那個 **Pairing code**（例如 `ABC123`）

### 9.4 批准配對

回到電腦的 PowerShell（不是 ngrok 那個視窗），輸入：

```powershell
clawdbot pairing approve line ABC123
```

（把 `ABC123` 換成你收到的配對碼）

會顯示 `Approved line sender Uxxxxxxxx`。

### 9.5 測試

再用 LINE 傳一次訊息：

```
你好
```

機器人應該會正常回覆了！

## 10. 常見問題與解決方法

### 問題 1：spawn npm ENOENT

**看到的錯誤訊息：**
```
Error: spawn npm ENOENT
```

**解決方法：**

1. 關掉 PowerShell
2. 重新開一個新的 PowerShell（系統管理員）
3. 輸入：
   ```powershell
   nvm use 24
   ```
4. 輸入：
   ```powershell
   setx PATH "$env:APPDATA\npm;$env:Path"
   ```
5. 關掉 PowerShell，再重新開一個
6. 再試一次原本的指令

---

### 問題 2：OAuth token 過期

**看到的錯誤訊息：**
```
OAuth token refresh failed for openai-codex
```

**解決方法：**

1. 在 PowerShell 輸入：
   ```powershell
   codex login
   ```
2. 瀏覽器會打開
3. 登入你的 ChatGPT/OpenAI 帳號
4. 登入完成後，回到 PowerShell
5. 輸入：
   ```powershell
   clawdbot gateway restart
   ```

---

### 問題 3：Node.js 版本太舊

**看到的錯誤訊息：**
```
required: { node: '>=24' }
```

**解決方法：**

```powershell
nvm install 24
nvm use 24
```

---

### 問題 4：ngrok 版本太舊

**看到的錯誤訊息：**
```
Your ngrok-agent version is too old
```

**解決方法：**

```powershell
ngrok update
```

---

### 問題 5：LINE 機器人沒回應

**可能原因 1：ngrok 已經關了**

解決：重新開 ngrok

```powershell
ngrok http 18789
```

然後去 LINE Developers 更新 Webhook URL（因為網址變了）

**可能原因 2：Clawdbot/Moltbot 沒在跑**

解決：

```powershell
clawdbot gateway start
```

**可能原因 3：還沒配對**

解決：看第 9 步的配對流程

## 12. 關閉與重新啟動

### 要關閉的時候

**第 1 步：關閉 ngrok**

在 ngrok 的 PowerShell 視窗按 `Ctrl + C`

**第 2 步：關閉 Clawdbot/Moltbot**

在另一個 PowerShell 視窗輸入：

```powershell
clawdbot gateway stop
```

---

### 下次要啟動的時候

**第 1 步：開 PowerShell**

點 **開始** → 搜尋 `PowerShell` → 打開

**第 2 步：啟動 Clawdbot/Moltbot**

```powershell
clawdbot gateway start
```

**第 3 步：開另一個 PowerShell 視窗，啟動 ngrok**

```powershell
ngrok http 18789
```

**第 4 步：更新 LINE Webhook（很重要！）**

因為 ngrok 免費版每次網址都會變，所以：

1. 看 ngrok 的 **Forwarding** 那行，複製新網址
2. 去 LINE Developers
3. 進入你的 Channel → **Messaging API**
4. 點 **Webhook URL** 的 **Edit**
5. 貼上新網址，後面加 `/line/webhook`
6. 點 **Update**

**第 5 步：測試**

用 LINE 傳訊息給機器人，看看有沒有回應

## 13. 進階：自動開發 + 部署 + 推送到 GitHub

### 13.1 事前準備

在開始之前，請確認你有：

| 項目 | 說明 |
|------|------|
| GitHub 帳號 | 用來存放程式碼 |
| Git 已安裝 | 版本控制工具 |
| Claude Code 已安裝 | 就是你剛裝好的龍蝦 |

---

### 13.2 安裝 Git（如果還沒裝）

在 PowerShell 輸入：

```powershell
winget install Git.Git
```

安裝完成後，**關掉 PowerShell 再重新開一個**。

確認安裝成功：

```powershell
git --version
```

看到版本號就代表成功了！

---

### 13.3 設定 Git 基本資訊

在 PowerShell 輸入（把名字和 Email 換成你的）：

```powershell
git config --global user.name "你的名字"
git config --global user.email "你的email@example.com"
```

---

### 13.4 安裝 GitHub CLI

GitHub CLI 讓你可以用指令操作 GitHub。

在 PowerShell 輸入：

```powershell
winget install GitHub.cli
```

安裝完成後，**關掉 PowerShell 再重新開一個**。

確認安裝成功：

```powershell
gh --version
```

---

### 13.5 登入 GitHub

在 PowerShell 輸入：

```powershell
gh auth login
```

會出現一系列問題，照著選：

1. **What account do you want to log into?**
   - 選 `GitHub.com`，按 Enter

2. **What is your preferred protocol for Git operations?**
   - 選 `HTTPS`，按 Enter

3. **Authenticate Git with your GitHub credentials?**
   - 選 `Yes`，按 Enter

4. **How would you like to authenticate GitHub CLI?**
   - 選 `Login with a web browser`，按 Enter

5. 瀏覽器會自動打開
   - 登入你的 GitHub 帳號
   - 輸入畫面上顯示的驗證碼
   - 點 **Authorize**

看到 `Logged in as 你的帳號名` 就代表成功！

### 13.6 使用 Claude Code 自動開發項目

Updates were rejected because the remote contains work that you do not have locally

先拉取再推送：

git pull --rebase
git push
```

#### 情境 20：請龍蝦幫你審查程式碼

**你說：**
```
幫我看這段程式碼有沒有問題：
def login(username, password):
    query = "SELECT * FROM users WHERE username='" + username + "' AND password='" + password + "'"
    result = db.execute(query)
    return result
```

**龍蝦會回覆：**
```
🚨 嚴重安全漏洞警告！

【問題分析】

❌ SQL Injection（SQL 注入攻擊）
這段程式碼有嚴重的資安漏洞！

【攻擊範例】
如果有人輸入：
- username: admin
- password: ' OR '1'='1

實際執行的 SQL 會變成：
SELECT * FROM users WHERE username='admin' AND password='' OR '1'='1'

結果：'1'='1' 永遠為真，駭客不用密碼就能登入！

【正確寫法】

✅ 使用參數化查詢（Parameterized Query）：

```python
def login(username, password):
    query = "SELECT * FROM users WHERE username = %s AND password = %s"
    result = db.execute(query, (username, password))
    return result
```

✅ 或使用 ORM（如 SQLAlchemy）：

```python
def login(username, password):
    user = User.query.filter_by(
        username=username,
        password=password
    ).first()
    return user
```

【額外建議】

1. 密碼不要存明碼！
   - 使用 bcrypt 或 argon2 加密

2. 加入登入失敗次數限制
   - 防止暴力破解

3. 使用 HTTPS
   - 保護傳輸過程

【修正後完整範例】
```python
import bcrypt

def login(username, password):
    # 參數化查詢，防止 SQL 注入
    query = "SELECT * FROM users WHERE username = %s"
    user = db.execute(query, (username,)).fetchone()

    if user is None:
        return None

    # 驗證加密後的密碼
    if bcrypt.checkpw(password.encode(), user['password_hash']):
        return user
    return None
```

⚠️ 資安很重要，這種漏洞可能讓整個資料庫被偷走！
```

**效果：** 找出程式漏洞，還教你怎麼修！
